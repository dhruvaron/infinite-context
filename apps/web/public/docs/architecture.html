Continuum v1 · local-first architecture

One timeline. Finite model context. Durable local memory.

Continuum does not make a model context window literally infinite. It keeps the conversation addressable on this Mac, compiles inspectable memory, and selects a bounded evidence packet for each response.

Source of truth

Verbatim conversation events and original attachment bytes remain authoritative. Claims, topic pages, embeddings, and graph edges are rebuildable projections. A compiled statement should always lead back to supporting evidence.

Retrieval

Each turn combines recent conversation, full-text and vector candidates, graph neighbors, temporal relevance, explicit pins, and user-authored pages. A recorded retrieval trace explains what was selected, excluded, and budgeted.

Local and cloud boundary

Kept on this Mac

The complete timeline, wiki, graph, indexes, attachments, operational metadata, backups, and provider key reference remain local. The provider key itself is held in macOS Keychain.

Sent for a cloud response

The current request, a bounded set of recent turns, selected evidence, and tool instructions are sent to the configured provider with provider storage disabled.

Integrity and recovery

  • Mutations are idempotent and destructive actions calculate their cascade first.
  • Attachments are content-addressed and must finish extraction before a message references them.
  • Managed backups and portable, checksummed exports support recovery.
  • User-authored topic revisions are trusted and are not silently overwritten.

Future coding-agent boundary

V1 can authorize read-only workspace roots, but it cannot modify repositories. Future coding scopes can add sandboxed tools, branches, and multi-agent work while preserving the same event, provenance, and revision model.
